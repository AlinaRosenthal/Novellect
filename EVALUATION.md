## B. Технологии

### 5.1 RAG
- локальный индекс + retrieval + grounded answer по найденным источникам.

### 5.2 Агент
- реализован single-agent full-режим: планирование подзапросов, запуск поисков и сбор grounded-ответа.

### 5.3 Инференс модели без API
- локальная LLM через `transformers`;
- внешние API для ответа модели не используются.

### 5.4 Дообучение модели
- добавлен отдельный pipeline адаптации embedding-модели по собственной библиотеке;
- добавлена параметро-эффективная LoRA-адаптация с merge после обучения для стабильного инференса;
- адаптация выполняется отдельно и не ухудшает базовый режим по умолчанию.

### 5.5 Оптимизация под слабое железо
- сохранён и выделен отдельный `Lite` режим;
- он не зависит от локальной генеративной LLM и подходит для CPU-only.

## C. Надёжность

### 6.1 Обработка сложных / неопределённых запросов

- есть сообщение о необходимости уточнения, если запрос слишком общий.

### 6.2 Анализ вычислительных потребностей
- добавлен блок в UI с оценкой требований для `Lite` и `Full`.
