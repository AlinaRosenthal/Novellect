# Novellect merged build

Эта сборка объединяет две ветки проекта в один рабочий пакет.

Что внутри:
- корректный routing запросов: `title / entity / specific / keyword / recommendation`
- book-level ранжирование вместо шумного chunk-only top-N
- дедупликация одинаковых книг в выдаче
- точные сниппеты вокруг совпадений
- ручной и автоматический режимы `auto / cpu / gpu`
- локальная загрузка, ZIP, Google Drive, Google Cloud Storage
- fallback без `sentence-transformers`: если локальная модель не загрузилась, поиск остаётся рабочим через lexical/TF-IDF контур

## Быстрый старт

```bash
pip install -r requirements.txt
streamlit run app.py
```

## После обновления проекта

Если у вас был старый индекс, лучше пересобрать его:

```bash
rm -f storage.json vector_db.npz search_cache.pkl
```

Затем заново загрузить датасет.

## Что изменено по качеству

- убрано вредное глобальное query expansion
- короткие запросы больше не уезжают сразу в общий semantic режим
- title-like запросы обрабатываются отдельно
- entity/specific запросы идут в lexical-first retrieval
- финальная выдача собирается по книгам, а не по чанкам
- `last_opened` и `open_count` не участвуют в релевантности поиска

## Что проверять после переиндексации

Обязательный smoke-test:
- `Доктор Живаго`
- `Преступление и наказание`
- `Отцы и дети`
- `Раскольников`
- `Базаров`
- `что-то мрачное и тревожное`
- `хочу книгу про конфликт поколений`

## Важное замечание

Если в старой папке проекта уже лежал `storage.json`, который не совпадает с датасетом, старый индекс будет давать мусор даже при хорошем коде. Для чистой проверки индекс нужно пересобрать.
