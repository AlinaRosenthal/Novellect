# 📚 Novellect: Универсальная система поиска книг

**Novellect** — это интеллектуальная локальная библиотека с мультиагентной архитектурой и универсальным анализом текста. В отличие от стандартного поиска, который ищет точные совпадения слов, Novellect использует **нейросетевые эмбеддинги** для поиска по смыслу и контексту, а также понимает настроение, стиль, сюжетные элементы и атмосферу произведений.

---

## ✨ Основные функции

- **Умный поиск (Hybrid Search):** Использует BM25 и нейросетевые эмбеддинги для поиска по смыслу и контексту.
- **Мультиагентная система:** 4 специализированных агента для анализа, поиска, ранжирования и ответов.
- **Универсальный анализ текста:** Автоматическое определение характеристик книг (настроение, стиль, сюжет, атмосфера).
- **Поддержка форматов:** Полноценный парсинг файлов `.pdf`, `.fb2`, `.epub` и `.txt`.
- **Семантическое разбиение (Semantic Chunking):** Алгоритм анализирует текст и разбивает его на смысловые блоки, сохраняя контекст повествования.
- **Локальное векторное хранилище:** Все данные (векторы и метаданные) хранятся локально, обеспечивая приватность и высокую скорость работы.
- **Интерактивный UI:** Удобный интерфейс на базе Streamlit с визуализацией прогресса индексации и результатов.

---

## 🛠 Технологический стек

- **Интерфейс:** [Streamlit](https://streamlit.io/)
- **AI Модель:** [Sentence-Transformers](https://www.sbert.net/) (`paraphrase-multilingual-MiniLM-L12-v2`)
- **BM25:** rank_bm25 с предварительной «умной» токенизацией и очисткой от стоп-слов
- **Обработка данных:** NumPy, Scikit-learn
- **Парсинг документов:** BeautifulSoup4, PyPDF2, Ebooklib, Chardet

## 🏗 Архитектура проекта

### Мультиагентная система

Приложение построено на модульной мультиагентной архитектуре:

| Агент | Назначение |
|-------|------------|
| **QueryAnalyzerAgent** | Анализирует запрос пользователя, определяет тип (vague/specific/general), извлекает ключевые слова, жанры, **настроение, стиль и сюжетные элементы** |
| **RetrievalAgent** | Выполняет гибридный поиск (семантика + BM25) с адаптивным балансом |
| **RankingAgent** | Ранжирует результаты: для vague запросов — по давности открытия, для запросов с характеристиками — многокритериальное ранжирование |
| **ResponseAgent** | Формирует ответ, группирует по книгам, отображает соответствие характеристикам |

### Компоненты системы

#### 1. Конвертер данных (`converter.py`)
Модуль отвечает за чтение книг разных форматов и превращение их в чистый текст:
* **TXT:** Автоматическое определение кодировки через `chardet`
* **FB2:** Парсинг XML-структуры, извлечение метаданных и текста
* **PDF:** Извлечение текстового слоя через PyPDF2
* **EPUB:** Извлечение содержимого через ebooklib, очистка HTML-тегов

#### 2. Поисковой движок (`search_engine.py`)
Реализует логику гибридного поиска и анализа текста:
* **Модель:** Предобученная нейросеть `paraphrase-multilingual-MiniLM-L12-v2` (понимает 50+ языков)
* **Семантическое разбиение:** Анализ косинусного сходства между соседними предложениями для создания смысловых чанков
* **BM25:** Классический алгоритм для точных совпадений (имена героев, специфические термины)
* **Гибридный поиск:** Комбинация семантических и BM25 оценок с коэффициентом alpha
* **Универсальный анализ текста:** Автоматическое определение характеристик по множеству категорий

**Анализируемые характеристики:**

| Категория | Характеристики |
|-----------|----------------|
| **Настроение** | позитивное, негативное, нейтральное, тревожное, романтичное |
| **Стиль** | диалоговый, описательный, динамичный, медленный |
| **Сюжет** | преодоление, борьба, путешествие, отношения, тайна, приключение |
| **Персонажи** | герой, антигерой, злодей, жертва |
| **Атмосфера** | таинственная, напряженная, уютная, мрачная, легкая |
| **Тон** | ироничный, серьезный, философский, эмоциональный |

#### 3. Пользовательский интерфейс (`app.py`)
Веб-интерфейс на базе Streamlit:
* **Sidebar:** Управление библиотекой, загрузка книг, статистика, информация о характеристиках книг
* **Main Area:** Поисковая строка с примерами запросов, выдача результатов
* **Результаты:** Карточки книг с указанием релевантности, сниппетами и соответствием характеристикам запроса

## 🌊 User Flow (Путь пользователя)

1. **Загрузка:** Пользователь перетаскивает файлы в область загрузки
2. **Индексация:**
   - Проверка дубликатов
   - Парсинг → семантическое разбиение → генерация векторов
   - **Универсальный анализ текста** по всем категориям
   - Сохранение в `vector_db.npz` и `storage.json`
3. **Поиск:**
   - Ввод запроса (например, *"хочу юморную историю про преодоление"*)
   - Агентный анализ запроса → определение характеристик
   - Гибридный поиск → многокритериальное ранжирование
4. **Просмотр:** Результаты с указанием соответствия искомым характеристикам

## 🛠 Технические подробности

- **Semantic Similarity:** Реализована через `sklearn.metrics.pairwise.cosine_similarity`
- **Data Persistence:** Локальный NumPy-стек для векторов, JSON для метаданных
- **Context Awareness:** Система запоминает не только текст, но и принадлежность к книге, формат и **полный профиль характеристик**
- **Кэширование:** TXT-версии книг для быстрой повторной индексации, кэш частых запросов
- **Batch processing:** Групповая обработка предложений для ускорения
