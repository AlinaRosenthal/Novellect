# 📚 Novellect: Hybrid Search Engine

**Novellect** — это интеллектуальная система для работы с локальной библиотекой книг. В отличие от стандартного поиска, который ищет точные совпадения слов, Novellect позволяет использовать **нейросетевые эмбеддинги**, чтобы находить информацию по смыслу запроса.

---

## ✨ Основные функции

-   **Умный поиск (Hybrid Search):** Использует bm25 и нейросетевые эмбеддинги для поиска по смыслу и контексту.
-   **Поддержка форматов:** Полноценный парсинг файлов `.pdf`, `.fb2`, `.epub` и `.txt`.
-   **Семантическое разбиение (Semantic Chunking):** Алгоритм анализирует текст и разбивает его на смысловые блоки, сохраняя контекст повествования.
-   **Локальное векторное хранилище:** Все данные (векторы и метаданные) хранятся локально, обеспечивая приватность и высокую скорость работы.
-   **Интерактивный UI:** Удобный интерфейс на базе Streamlit с визуализацией прогресса индексации и результатов.

---

## 🛠 Технологический стек

-   **Frontend:** [Streamlit](https://streamlit.io/)
-   **AI Модель:** [Sentence-Transformers](https://www.sbert.net/) (`paraphrase-multilingual-MiniLM-L12-v2`)
-   **BM25:** rank_bm25 с предварительной «умной» токенизацией и очисткой от стоп-слов
-   **Обработка данных:** NumPy, Scikit-learn
-   **Парсинг документов:** BeautifulSoup4, PyPDF2, Ebooklib, Chardet

## 🏗 Архитектура проекта

Приложение построено на модульной архитектуре, разделяющей обработку данных, поисковую логику и интерфейс.

### 1. Конвертер данных (`converter.py`)
Этот модуль отвечает за «чтение» книг разных форматов и превращение их в чистый текст.
*   **TXT:** Автоматически определяет кодировку (UTF-8, Windows-1251 и др.) через `chardet`.
*   **FB2:** Парсит XML-структуру, извлекая метаданные (название) и текстовые блоки (`<p>`, `<subtitle>`).
*   **PDF:** Использует `PyPDF2` для извлечения текстового слоя.
*   **EPUB:** Извлекает содержимое через `ebooklib`, очищая HTML-теги с помощью `BeautifulSoup`.

### 2. Поисковой движок (`search_engine.py`)
Реализует логику **Vector Search** (Векторного поиска):
*   **Модель:** Используется предобученная нейросеть `paraphrase-multilingual-MiniLM-L12-v2`. Она понимает 50+ языков, включая русский.
*   **Семантическое разбиение (Semantic Splitting):** Вместо нарезки текста на равные куски, алгоритм анализирует косинусное сходство между соседними предложениями. Если смысл резко меняется (ниже порога 0.6), создается новый «чанк» (смысловой фрагмент). Это сохраняет контекст при поиске.
*   **BM25:** Классический алгоритм ранжирования, основанный на частоте слов. Он находит точные совпадения: редкие имена героев, названия мест или специфические термины, которые нейросеть может «сгладить».
*   **RRF (Reciprocal Rank Fusion) & Weighting**: Результаты обоих алгоритмов нормализуются и объединяются с коэффициентом alpha (по умолчанию 0.7 в пользу семантики), что дает максимально точную выдачу.
*   **Хранение:**
    *   `vector_db.npz` — сжатый файл NumPy, хранящий векторы (числовые «отпечатки» смысла) и метаданные чанков.
    *   `storage.json` — реестр проиндексированных книг для быстрого отображения в интерфейсе.

### 3. Пользовательский интерфейс (`app.py`)
Веб-интерфейс на базе **Streamlit**:
*   **Sidebar:** Управление библиотекой, загрузка новых книг, отображение статистики и кнопка полной очистки базы.
*   **Main Area:** Поисковая строка с мгновенной выдачей результатов.
*   **Результаты:** Выводятся в виде раскрывающихся карточек (expanders) с указанием степени сходства (similarity score) и наиболее релевантного фрагмента текста.

## 🌊 User Flow (Путь пользователя)

1.  **Загрузка:** Пользователь перетаскивает файлы (.pdf, .fb2, .txt или .epub) в область загрузки.
2.  **Индексация:**
    *   Приложение проверяет, не была ли книга добавлена ранее (по названию).
    *   Происходит парсинг → семантическое разбиение → генерация векторов.
    *   Данные дописываются в локальные файлы `.npz` и `.json`.
3.  **Поиск:**
    *   Пользователь вводит запрос (например, *"философия и смысл жизни"* или *"описание далеких планет"*).
    *   Система векторизует запрос и сравнивает его со всеми чанками в базе.
4.  **Просмотр:** Пользователь получает топ-3 самых близких по смыслу отрывка из всей своей библиотеки.

---

## 🛠 Технические подробности

*   **Semantic Similarity:** Реализована через `sklearn.metrics.pairwise.cosine_similarity`.
*   **Data Persistence:** Вместо тяжелых баз данных (Vector DB) используется локальный NumPy-стек, что делает прототип переносимым и быстрым на небольших библиотеках.
*   **Context Awareness:** Система запоминает не только текст, но и принадлежность к книге и её формат.
